# Анализ продуктовых метрик маркетплейса

В этом проекте я разобрал данные маркетплейса и посмотрел на основные метрики: как пользователи покупают, сколько приносят денег и есть ли проблемные места.

## Что сделал

Сначала подготовил данные в Python:
- загрузил таблицы
- проверил дубликаты и пропуски
- привёл даты к нужному формату
- исправил некорректные значения
- отфильтровал данные по нужному периоду

После этого работал уже в SQL и считал метрики.

## Какие метрики посчитал

- топ категорий по количеству отменённых заказов  
- топ пользователей по выручке  
- средний чек (AOV)  
- среднее количество товаров в заказе  
- DAU (активные пользователи)  
- ARPPU  
- retention  
- LTV  

## Что можно понять из анализа

- есть категории с повышенным количеством отмен — это потенциальные проблемы  
- небольшая группа пользователей даёт основную выручку  
- поведение пользователей можно оценить через средний чек и количество товаров  
- retention и LTV помогают понять, насколько пользователи возвращаются и сколько приносят в долгую
